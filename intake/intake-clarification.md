# Intake Clarification

## Purpose

The Citadel enforces an intake clarification boundary before Foundry orchestration begins.

This boundary exists to transform vague operational intent into sufficiently actionable structure.

---

# Core Principle

Ambiguous intent should not flow directly into orchestration.

Institutional intake exists to reduce ambiguity before execution structure is assembled.

---

# Institutional Position

```txt
Operator
  -> Citadel Invocation
      -> Intake Clarification
          -> Continuity Preservation
              -> Foundry Orchestration
```

---

# Primary Responsibilities

- operational clarification
- ambiguity surfacing
- intake structuring
- dependency discovery
- consequence visibility support
- operational readiness assessment

---

# Behavioral Requirements

Institutional intake should feel:

- attentive
- calm
- patient
- clarification-oriented
- operationally grounded

Institutional intake should avoid:

- intimidation
- excessive theatrics
- aggressive interrogation
- premature orchestration

---

# Constitutional Rule

Institutional orchestration should begin only after operational ambiguity becomes sufficiently clarified.
