# Citadel Memory Doctrine

## Purpose

Memory Doctrine defines how operational memory is classified, preserved, reconstructed, and governed inside The Citadel.

The goal is continuity without uncontrolled accumulation.

---

# Core Principle

Conversation is not memory.

Memory is not truth.

Memory is governed operational continuity.

---

# Memory Classes

## Ephemeral Memory

Short-lived working context.

Examples:
- temporary reasoning
- active conversational context
- transient runtime state

Ephemeral memory is expected to decay.

## Operational Memory

Information required to continue active operations coherently.

Examples:
- active mission state
- staffing assignments
- restoration state
- unresolved escalations
- runtime configuration

## Institutional Memory

Long-duration operational knowledge preserved for continuity.

Examples:
- doctrine
- governance rules
- architecture decisions
- validated workflows
- constitutional principles

## Historical Memory

Preserved for reconstruction, lineage, or audit.

Historical retention does not imply operational validity.

---

# Memory Sources

Citadel memory may originate from:

- artifacts
- operational logs
- runtime state
- verification records
- restoration history
- governance decisions
- escalation chains
- human directives

Artifacts remain the preferred durable memory surface.

---

# Memory Authority

Not every actor may alter institutional memory.

Memory mutation authority depends on:

- role
- consequence tier
- artifact trust state
- governance constraints
- restoration status

Unverified actors may propose memory updates.

They may not silently institutionalize them.

---

# Memory Persistence Rules

## Ephemeral Memory

May expire naturally.

## Operational Memory

Must persist until:

- operation completion
- escalation resolution
- restoration completion
- explicit archival

## Institutional Memory

Requires:

- provenance
- lineage
- reviewability
- restoration traceability

---

# Memory Reconstruction

The Citadel should be capable of reconstructing:

- what happened
- why it happened
- who authorized it
- what evidence existed
- what verification occurred
- what uncertainty remained
- what final disposition was assigned

If operational reconstruction is impossible, continuity is compromised.

---

# Memory Drift

Long-running systems accumulate memory drift.

Examples:

- obsolete assumptions
- stale operational context
- superseded doctrine
- forgotten constraints
- silently inherited behavior

Memory must therefore support:

- supersession
- deprecation
- archival
- restoration
- lineage tracing

---

# Memory and Truth

Memory persistence does not prove correctness.

Institutional memory may contain:

- outdated doctrine
- incorrect assumptions
- failed approaches
- deprecated operational patterns

Trust requires verification, not persistence.

---

# Human Memory Interaction

Humans may:

- introduce memory
- override memory
- archive memory
- deprecate memory
- restore memory

Human intervention must still preserve lineage and operational traceability.

---

# Constitutional Rule

A system that cannot preserve coherent memory cannot sustain coherent governance.
