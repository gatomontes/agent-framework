# Epistemic Contract

## Status

Canonical AFW contract.

This document defines how information enters, propagates, transforms, conflicts, decays, and becomes trusted within AFW-governed operations.

The Epistemic Contract governs:
- external knowledge ingress
- evidence classification
- uncertainty handling
- provenance tracking
- contradiction management
- freshness semantics
- knowledge auditability
- institutional truth discipline

---

# Purpose

AFW requires epistemic governance because operational correctness depends upon informational integrity.

Without epistemic governance:
- false information contaminates reasoning
- invalid assumptions propagate silently
- verification collapses into performance
- authority decisions become corrupted
- restoration becomes impossible
- institutional drift accelerates

Operational governance without epistemic governance is structurally unstable.

---

# Core Principle

No information becomes institutional truth merely because:
- it was stated confidently
- a runtime produced it
- a source appears authoritative
- it aligns with expectations
- it is convenient
- it is popular
- it was previously accepted

Institutional trust must remain evidence-bound, auditable, and revisable.

---

# Epistemic States

Information inside AFW must remain classifiable.

Minimum epistemic states:
- observed
- verified
- probable
- plausible
- speculative
- contradicted
- unverifiable
- superseded
- stale
- adversarial

Information may transition between states as evidence changes.

---

# Source Classes

All meaningful information ingress should preserve source classification.

Minimum source classes:

## Primary

Original authoritative source.

Examples:
- official document
- direct measurement
- raw logs
- firsthand observation
- source repository

## Secondary

Interpretation or summary of primary material.

Examples:
- articles
- analyses
- reviews
- summaries

## Synthetic

Generated by a runtime, model, simulation, or transformation process.

Synthetic output is not automatically false.

Synthetic output is not automatically verified.

## Inferred

Derived through reasoning from available evidence.

Inference strength must remain explicit.

## Unverified

Information lacking sufficient validation.

## Adversarial

Information potentially manipulated, deceptive, poisoned, malicious, or strategically distorted.

Adversarial classification does not imply certainty of falsehood.

It indicates elevated epistemic risk.

External information ingress is governed activity.

Ingress includes:
- web retrieval
- documents
- repositories
- APIs
- human reports
- model outputs
- databases
- logs
- measurements
- screenshots

---

# Trust Semantics

Trust is not binary.

Trust assessment should consider:
- provenance quality
- source independence
- corroboration
- reproducibility
- freshness
- contradiction pressure
- transformation distance
- adversarial exposure
- verification depth

Confidence does not replace verification.

---

# Corroboration

High-impact claims should seek corroboration.

Corroboration should favor:
- independent sources
- independent retrieval paths
- independent methodologies
- direct evidence over repetition

Consensus is not proof.

Repeated unsupported claims do not become verified through repetition.

---

# Contradiction Handling

Contradictions are first-class epistemic events.

Contradictions must not be hidden for convenience.

Contradiction states include:
- unresolved
- partially resolved
- resolved
- structurally disputed

Contradictions may trigger:
- restoration
- escalation
- verification replay
- topology reassessment
- authority review
- mission suspension

---

# Uncertainty Discipline

Uncertainty must survive transmission.

No actor may collapse:
- speculative
into
- verified

through tone, implication, omission, or confidence theater.

Outputs should distinguish:
- verified claims
- probable claims
- unverified claims
- contradictions
- gaps in evidence
- unresolved uncertainty

---

# Freshness Semantics

Information decays over time.

Knowledge freshness should consider:
- source age
- environmental change
- dependency change
- mission context change
- governance updates
- contradiction emergence

Stale information may require:
- revalidation
- corroboration replay
- dependency invalidation
- restoration

---

# Epistemic Drift

Epistemic drift occurs when institutional belief gradually diverges from evidence quality.

Signals include:
- confidence inflation
- source laundering
- citation theater
- unresolved contradictions ignored
- stale assumptions treated as current
- synthetic outputs treated as verified facts
- repeated inference mistaken for observation
- convenience replacing verification

Epistemic drift is governance corruption.

---

# Runtime Separation

AFW distinguishes between:
- runtime memory
- mission memory
- institutional memory
- verified knowledge
- speculative working state
- synthetic reasoning artifacts

Runtime-generated content must not silently enter institutional memory as verified truth.

---

# Verification Interface

Verification systems must preserve epistemic state.

Verification must not:
- erase uncertainty
- suppress contradictions
- hide provenance
- compress evidence state into theatrical certainty

Verification without provenance preservation is incomplete.

---

# Restoration Interface

Restoration processes may require:
- knowledge rollback
- contradiction replay
- source reevaluation
- stale dependency invalidation
- provenance reconstruction
- audit reconstruction

Epistemic restoration must remain auditable.

---

# Authority Interface

Authority does not determine truth.

Governance actors may:
- prioritize
- halt
- escalate
- classify
- authorize action

They may not redefine evidence quality through authority alone.

Institutional override authority does not automatically rewrite epistemic state.

---

# Audit Requirements

Epistemic audit should reconstruct:
- source origin
- retrieval path
- transformation chain
- contradiction history
- freshness state
- verification path
- unresolved uncertainty
- dependent decisions

Knowledge must remain reconstructable.

---

# Noncompliance

An actor violates epistemic doctrine if it:
- fabricates certainty
- hides contradictions
- launders unverifiable claims into verified state
- suppresses provenance
- treats synthetic output as automatically authoritative
- ignores freshness decay
- conceals uncertainty
- collapses verification into persuasion
- uses popularity as truth substitute

Epistemic noncompliance requires review and possible restoration.
