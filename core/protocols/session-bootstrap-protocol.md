# Citadel Session Bootstrap Protocol

## Purpose

The Session Bootstrap Protocol defines how a runtime, wrapper, or operator should initialize a Citadel-governed session before active execution begins.

Bootstrap exists to prevent incoherent starts, hidden context loss, and improvisational continuation.

---

# Core Principle

A session should not begin by guessing where continuity lives.

Bootstrap must load governing context before execution pressure starts distorting judgment.

---

# Bootstrap Order

When a runtime or operator begins work, load only enough context to establish coherent operation in this order:

1. active role and authority context
2. user or mission-owner context when relevant
3. current mission or task packet
4. recent operational continuity artifacts
5. longer-duration memory only when authority and context permit it

Conversation history alone is not a sufficient bootstrap surface when governed artifacts already exist.

---

# Preferred Bootstrap Sources

Prefer:

- active task packets
- mission artifacts
- authority packets
- runtime reports
- restoration packets
- recent operational logs
- curated institutional memory

Prefer artifacts over recollection.

Prefer recent operational continuity over broad historical loading.

---

# Memory Gating

Not all memory should be loaded into every context.

Long-duration or sensitive memory should be loaded only when:

- the session has authority to access it
- the context requires it
- the risk of disclosure is acceptable

Shared, external, or multi-party contexts require stricter memory gating than private internal work.

---

# Bootstrap Constraints

Bootstrap may not:

- load sensitive memory by reflex
- infer authority from available files
- treat stale historical memory as current truth
- bypass active mission artifacts because conversational context feels easier

If the correct bootstrap surface is unclear, the runtime should escalate or proceed with explicit continuity uncertainty.

---

# Bootstrap Minimum Output

A coherent bootstrap should leave the session able to state:

- what operation is active
- who or what owns authority
- what continuity artifacts are controlling
- what remains uncertain
- whether the session is ready to proceed

---

# Constitutional Rule

Improvised session startup is a continuity failure.
