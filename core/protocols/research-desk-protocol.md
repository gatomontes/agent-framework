# Research Desk Protocol

## Status

Canonical Citadel protocol.

This protocol governs research missions, evidence collection, synthesis release, and adversarial review inside Citadel-governed operations.

Research is not casual search.

Search is input.

Release is a governed decision.

---

# Purpose

The Research Desk Protocol exists to prevent:
- search being mistaken for truth
- synthesis being released without release criteria
- deadline pressure silently lowering evidence standards
- unverified claims laundering themselves into doctrine
- collection without scope discipline
- publication of unstable conclusions

---

# Core Principle

Research must declare scope and release threshold before evidence collection begins.

If time pressure increases, narrow the scope.

Do not lower the evidence standard without explicit authority.

---

# Research Desk Stages

## 1. Scope Definition

Before active collection begins, the desk should declare:
- research question
- decision relevance
- scope boundaries
- excluded areas
- required evidence standard
- release threshold
- time constraint, if any

Undefined scope is not research readiness.

## 2. Discovery

Discovery locates candidate sources, entities, documents, timelines, and search angles.

Discovery does not grant trust.

Outputs may include:
- source shortlist
- search log
- candidate evidence map
- open questions

## 3. Verification

Verification tests whether candidate sources and claims are fit for use.

Verification should assess:
- provenance
- authorship
- date
- independence
- contradiction exposure
- relevance
- trustworthiness

## 4. Extraction

Extraction pulls durable evidence from verified material.

Extraction should preserve:
- exact claim
- source reference
- date context
- caveats
- uncertainty

## 5. Synthesis

Synthesis organizes verified evidence into an operationally useful form.

Synthesis must separate:
- verified claims
- probable claims
- disputed claims
- unknowns
- blocked questions

## 6. Adversarial Review

High-consequence research should undergo adversarial review before release.

Adversarial review pressures:
- unsupported claims
- hidden assumptions
- contradiction omission
- overstatement
- narrative drift

## 7. Release Decision

Research may be released only when the evidence still supports the conclusion after required review.

Possible outcomes:
- release
- release with caveats
- hold for more verification
- narrow claim set
- terminate as inconclusive

---

# Deadline Rule

Time pressure may:
- narrow question scope
- reduce breadth
- shorten collection window
- defer lower-priority subquestions

Time pressure must not:
- convert unverified claims into verified claims
- suppress contradiction reporting
- erase uncertainty
- lower release threshold by convenience

---

# Release Threshold

The release threshold should be explicit enough to audit.

It may include:
- minimum source quality
- minimum corroboration
- contradiction tolerance
- freshness requirement
- review requirement
- consequence-sensitive confidence floor

If the threshold changes during the mission, the change must be recorded.

---

# Research Artifact Expectations

Research missions should emit durable artifacts such as:
- discovery logs
- source verification notes
- extracted evidence scrolls
- synthesis briefs
- adversarial review notes
- release decisions

Conversation alone is not adequate institutional research memory.

---

# Constitutional Rule

Research is a quality system.

Collection is not conclusion.

Synthesis is not release.
