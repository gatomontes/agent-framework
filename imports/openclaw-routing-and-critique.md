# OpenClaw Routing And Critique

## What Survives

### 1. Critique as a distinct operating role

Useful pattern from `blackquill`:

- critique is not politeness
- contradiction pressure should be explicit
- weak assumptions should be attacked, not massaged

Citadel-worthy translation:

- critical review roles should have permission to surface contradiction without social smoothing
- critique should be structurally distinct from execution and synthesis

What does not survive:

- “no guardrails” posturing
- theatrical brutality as identity

Reason:

The useful part is epistemic aggression, not adolescent edginess.

### 2. Intake normalization before delegation

Useful pattern from `project`:

- turn vague requests into explicit task objects
- classify task type before choosing workers
- choose the smallest useful execution shape
- define merge ownership and output contracts before dispatch

Citadel-worthy translation:

- intake should normalize the task before orchestration
- delegation should be shaped by artifact/output requirements, not by delegation reflex
- every multi-worker plan needs explicit merge ownership

### 3. Output contracts per worker

Useful pattern:

- every worker returns something bounded and mergeable
- “thoughts” are not enough when synthesis is expected

Citadel-worthy translation:

- delegated roles should have explicit output contracts
- contradiction between outputs should be surfaced, not blurred into fake agreement

### 4. Evidence-separated research behavior

Useful pattern from `researcher`:

- separate verified, probable, unverified, contradictory, and unknown
- preserve audit trail and provenance
- treat anomalies as signals, not clutter

Citadel-worthy translation:

- evidence classes should remain explicit
- uncertainty and contradiction need first-class preservation
- primary-source preference should remain institutional

### 5. Persona building from resolved upstream structure

Useful pattern from `persona-builder`:

- downstream builders should trust resolved upstream structure by default
- do not silently resolve unresolved slots
- preserve handoff responsibilities and escalation gaps

Citadel-worthy translation:

- downstream synthesis should not redo upstream governance unless the handoff is unsafe
- persona/spec creation should preserve role boundaries and unresolved uncertainty

## What Does Not Survive As Doctrine

- Bureau-specific naming like `Vesper`
- “Sunshine” critique language
- local role mythology
- presentation-heavy XML ritual when lighter schemas would do

Reason:

The structure matters. The costume does not.

## Recommended Citadel Follow-Ons

- formalize a generic intake-to-routing object in Citadel terms
- define merge ownership as an explicit part of delegation doctrine
- preserve critique as a constitutional role with bounded but strong contradiction authority
