# Composition Rules

Use this checklist to stay inside the selected local UI kit.

## Required Checks

- Confirm the source root before editing.
- Refuse to begin UI work until the source path exists and is verified.
- Generate an inventory of the source tree before choosing templates.
- Verify each major section in the output maps to a real file, fragment, or repeated pattern in the source tree.
- Prefer whole local sections over hand-built approximations.
- Reuse the kit's existing asset paths, CSS classes, and JS hooks.

## Disallowed Moves

- Importing another framework, UI kit, CDN stylesheet, or icon library that is not already inside the source tree.
- Creating a brand-new visual language that does not resemble the local kit.
- Referencing remote stock assets when a comparable local asset already exists.
- Presenting speculative components as if they are part of the kit.
- Quietly inventing a missing component instead of calling out that it does not exist.

## Good Adaptations

- Combining two existing cards into a denser dashboard row.
- Swapping text, calls to action, images, and icon choices.
- Removing decorative layers, tabs, or counters from an existing block.
- Adjusting column widths or responsive visibility using conventions already present nearby.

## If The Kit Is Thin

- Choose the closest local pattern and adapt it conservatively.
- Call out the missing pattern explicitly.
- Keep the result structurally simple rather than inventing a new system.
- Pause for approval if the requested UI cannot be built honestly from the source tree.
