# Progress Note Template

Use this structure for the dated Markdown file. Adapt the headings if the project already has a strong documentation pattern, but keep the content categories intact.

```md
# Project Progress Review - <Project Name>

Date: YYYY-MM-DD
Source Brief: <path or file name>
Review Scope: <folder, repo, or project name>

## Executive Summary

2-4 sentences on current status, overall momentum, and whether the project is on track, partially complete, blocked, or drifting.

## Original Objectives

- Objective 1
- Objective 2
- Objective 3

## Current Progress

### Completed

- Item with evidence

### In Progress

- Item with evidence

### Not Started Or Unclear

- Item with evidence gap called out

## Scope Changes And Decisions

- Decision, change, or newly discovered deviation from the original brief

## Risks And Blockers

- Risk or blocker

## Recommended Next Steps

- Next action 1
- Next action 2

## Evidence Reviewed

- Brief: <file>
- Status docs: <files>
- Recent files or commits: <files or short notes>
```

Evidence should be specific enough that another person can retrace the reasoning quickly.
